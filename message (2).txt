Okay, let's break down Page Replacement Algorithms from scratch, just like we did with Banker's.

## Page Replacement Algorithms: The Core Problem

### The "Why": Virtual Memory and Limited RAM

1.  **Virtual Memory:** Modern operating systems create an illusion for each program that it has access to a huge, continuous block of memory (e.g., 4GB or more), even if the computer physically has less RAM. This "virtual memory" space is often much larger than the actual physical memory (RAM).
2.  **Physical Memory (RAM):** This is the actual, fast memory chips in your computer. It's finite and often much smaller than the sum of all virtual memory spaces of running programs.
3.  **Pages and Frames:**
    *   **Pages:** The virtual memory space of a program is divided into fixed-size blocks called "pages." (e.g., 4KB, 8KB).
    *   **Frames:** The physical memory (RAM) is also divided into fixed-size blocks of the *same size* as pages, called "frames."
    *   When a program runs, its pages are loaded into these physical memory frames.

### The Problem: RAM Gets Full!

*   Not all pages of a program can reside in RAM at the same time. The OS moves pages between RAM and the slower disk (hard drive/SSD).
*   When a program tries to access a piece of data or an instruction, the CPU translates the virtual address to a physical address.
*   **Page Hit:** If the required page is already in a physical memory frame, great! The program continues quickly.
*   **Page Fault (Miss):** If the required page is *not* in any physical memory frame, a "page fault" occurs.
    1.  The CPU halts the current process.
    2.  The OS takes over.
    3.  The OS finds the required page on the disk.
    4.  The OS loads that page from disk into an empty physical memory frame.
    5.  The CPU resumes the process.

### The Big Question for Page Replacement:

> **What happens if a page fault occurs AND all physical memory frames are already full?**

You can't just load the new page because there's no space. You *must* make space by **evicting (kicking out)** one of the pages currently in RAM to disk to make room for the new one.

**Which page do you kick out?** That's what Page Replacement Algorithms decide. The goal is to minimize page faults.

---

## Key Terms & Concepts for Page Replacement

*   **Page Reference String:** A sequence of page numbers that a program tries to access. This is our input for simulating algorithms.
    *   Example: `7, 0, 1, 2, 0, 3, 0, 4, 2, 3, 0, 3`
*   **Frames:** The number of available physical memory frames (RAM capacity).
*   **Page Fault:** Occurs when the requested page is not in memory.
*   **Hit:** Occurs when the requested page is already in memory.
*   **Goal:** Minimize the number of page faults.

---

## The Three Classic Page Replacement Algorithms

We'll look at:

1.  **FIFO (First-In, First-Out)**
2.  **LRU (Least Recently Used)**
3.  **Optimal (MIN/OPT)**

---

## 1. FIFO (First-In, First-Out)

### Philosophy:

> "The page that has been in memory the longest is replaced."

Think of it like a queue: the first page to enter memory is the first one to be evicted when space is needed. It doesn't care if the page is heavily used or not.

### How it Works:

1.  When a page is referenced:
    *   **Hit:** If the page is in memory, do nothing.
    *   **Page Fault:**
        *   If there are empty frames, load the page into an empty frame.
        *   If all frames are full, remove the **oldest** page (the one that entered memory first) and load the new page into its frame.

### Example Walkthrough:

*   **Page Reference String:** `7, 0, 1, 2, 0, 3, 0, 4, 2, 3, 0, 3`
*   **Frames:** 3

| Reference | Memory State (FIFO) | Page Fault? | Notes                                         |
| :-------- | :------------------ | :---------- | :-------------------------------------------- |
| 7         | `[7, -, -]`         | PF          | 7 loaded.                                     |
| 0         | `[7, 0, -]`         | PF          | 0 loaded.                                     |
| 1         | `[7, 0, 1]`         | PF          | 1 loaded. All frames full. Oldest is 7.       |
| 2         | `[0, 1, 2]`         | PF          | 7 is removed. 2 loaded. Oldest is 0.          |
| 0         | `[0, 1, 2]`         | Hit         | 0 already in memory.                          |
| 3         | `[1, 2, 3]`         | PF          | 0 is removed. 3 loaded. Oldest is 1.          |
| 0         | `[2, 3, 0]`         | PF          | 1 is removed. 0 loaded. Oldest is 2.          |
| 4         | `[3, 0, 4]`         | PF          | 2 is removed. 4 loaded. Oldest is 3.          |
| 2         | `[0, 4, 2]`         | PF          | 3 is removed. 2 loaded. Oldest is 0.          |
| 3         | `[4, 2, 3]`         | PF          | 0 is removed. 3 loaded. Oldest is 4.          |

| 3         | `[2, 3, 0]`         | Hit         | 3 already in memory.                          |

### Pros:
*   Simple to understand and implement.

### Cons:
*   Can suffer from **Belady's Anomaly**: Increasing the number of frames might *increase* page faults in some cases.
*   Ignores usage patterns; might evict a frequently used page if it's old.

### C++ Implementation Strategy:
*   Use an `unordered_set` for quick `O(1)` checks if a page is in memory.
*   Use a `queue` to maintain the order of pages in memory (oldest at the front). When a page needs to be removed, `pop()` from the queue. When a new page is added, `push()` to the queue.

## 2. LRU (Least Recently Used)

### Philosophy:

> "The page that has not been used for the longest period of time is replaced."

This is based on the idea that if a page hasn't been used recently, it's less likely to be used in the near future. This is generally a good heuristic.

### How it Works:

1.  When a page is referenced:
    *   **Hit:** If the page is in memory, update its "last used" timestamp/position to indicate it was just used.
    *   **Page Fault:**
        *   If there are empty frames, load the page into an empty frame.
        *   If all frames are full, remove the page that was used the **longest time ago** (the least recently used) and load the new page.

### Pros:
*   Generally performs better than FIFO (no Belady's Anomaly).
*   Utilizes the principle of locality (what's used recently is likely to be used again soon).

### Cons:
*   More complex to implement than FIFO (requires tracking usage time/order).
*   Actual implementation often involves hardware support (e.g., counters or reference bits) or software overhead (like maintaining a linked list or stack).

### C++ Implementation Strategy:
*   Use a `vector` to represent the pages currently in memory. The order in the `vector` signifies recency: `memory.begin()` is LRU, `memory.end()-1` is MRU.
*   When a page is hit: `find()` its position, `erase()` it, then `push_back()` it.
*   When a page fault occurs and memory is full: `erase(memory.begin())` (to remove LRU), then `push_back()` the new page.

## 3. Optimal (MIN / OPT)

### Philosophy:

> "Replace the page that will not be used for the longest period of time in the future."

This algorithm is clairvoyant – it knows the *entire future* page reference string. It's impossible to implement in a real OS, but it's used as a benchmark to compare the efficiency of other algorithms (no real algorithm can perform better than Optimal).

### How it Works:

1.  When a page is referenced:
    *   **Hit:** If the page is in memory, do nothing.
    *   **Page Fault:**
        *   If there are empty frames, load the page into an empty frame.
        *   If all frames are full, look into the *future* page reference string. Identify which page currently in memory will be used farthest in the future (or not at all). Evict that page.

### Example Walkthrough:

*   **Page Reference String:** `7, 0, 1, 2, 0, 3, 0, 4, 2, 3, 0, 3`
*   **Frames:** 3

| Reference | Memory State (Optimal) | Page Fault? | Notes                                                              |
| :-------- | :------------------- | :---------- | :----------------------------------------------------------------- |
| 7         | `[7, -, -]`          | PF          | 7 loaded.                                                          |
| 0         | `[7, 0, -]`          | PF          | 0 loaded.                                                          |
| 1         | `[7, 0, 1]`          | PF          | 1 loaded. All frames full.                                         |
| 2         | `[0, 1, 2]`          | PF          | Future: 7 (never used again), 0 (used next at ref 5), 1 (used at ref 8). Replace 7. |
| 0         | `[0, 1, 2]`          | Hit         | 0 already in memory.                                               |
| 3         | `[1, 2, 3]`          | PF          | Future: 0 (used next at ref 7), 1 (never used again), 2 (used at ref 9). Replace 1. |
| 0         | `[2, 3, 0]`          | Hit         | 0 already in memory.                                               |
| 4         | `[0, 2, 4]`          | PF          | Future: 2 (used next at ref 9), 3 (used at ref 10), 0 (used at ref 11). Replace 3. |
| 2         | `[0, 2, 4]`          | Hit         | 2 already in memory.                                               |
| 3         | `[0, 2, 3]`          | PF          | Future: 0 (used at ref 11), 2 (never used again), 4 (never used again). Replace 4. |
| 0         | `[0, 2, 3]`          | Hit         | 0 already in memory.                                               |
| 3         | `[0, 2, 3]`          | Hit         | 3 already in memory.                                               |
|           | **Total Page Faults:** **6**        |                                                                    |
**(Re-verified this trace against the code. The optimal example produces 6 page faults.)**

### Pros:
*   Achieves the lowest possible number of page faults for a given page reference string.
*   Serves as a benchmark for comparison.

### Cons:
*   Impossible to implement in real systems because it requires knowledge of the future.

### C++ Implementation Strategy:
*   When a page fault occurs and memory is full:
    *   Iterate through each page currently in memory.
    *   For each page, scan the *remaining part* of the `pages` reference string (from the current position onwards) to find its next use.
    *   Keep track of the page whose next use is farthest in the future (or if it's never used again, that's the one).
    *   Evict that page.

## C++ Implementation (All Three Combined)

```cpp
#include <iostream>
#include <vector>
#include <queue>
#include <unordered_set> // For O(1) average time lookup
#include <algorithm>     // For std::find, std::erase
#include <climits>       // For INT_MAX

using namespace std;

// ==================== FIFO (First-In, First-Out) ====================
int fifo(const vector<int>& pages, int frames) {
    unordered_set<int> s;       // Stores pages currently in memory for O(1) lookup
    queue<int> q;               // Stores pages in FIFO order (oldest at front)
    int pageFaults = 0;

    for (int page : pages) {
        // If page is not in memory (page fault)
        if (s.find(page) == s.end()) {
            pageFaults++;

            // If memory is full, remove the oldest page
            if ((int)s.size() == frames) {
                int oldest_page = q.front(); // Get the oldest page
                q.pop();                     // Remove it from the queue
                s.erase(oldest_page);        // Remove it from the set
            }

            // Add the new page to memory
            s.insert(page);
            q.push(page);
        }
        // If page is already in memory (hit), do nothing (FIFO doesn't care about hits)
    }
    return pageFaults;
}

// ==================== LRU (Least Recently Used) ====================
int lru(const vector<int>& pages, int frames) {
    // A vector acts as memory. Elements are ordered by recency.
    // Front = LRU, Back = MRU (Most Recently Used)
    vector<int> memory;
    int pageFaults = 0;

    for (int page : pages) {
        auto it = find(memory.begin(), memory.end(), page);

        // If page not in memory (page fault)
        if (it == memory.end()) {
            pageFaults++;

            // If memory is full, remove the LRU page (at the front)
            if ((int)memory.size() == frames) {
                memory.erase(memory.begin()); // Remove the least recently used
            }
            // Add the new page (becomes the most recently used)
            memory.push_back(page);
        } else {
            // If page is in memory (hit), move it to the MRU position (back)
            memory.erase(it);                  // Remove from its current position
            memory.push_back(page);            // Add to the end (most recently used)
        }
    }
    return pageFaults;
}

// ==================== Optimal (MIN / OPT) ====================
int optimal(const vector<int>& pages, int frames) {
    vector<int> memory;
    int pageFaults = 0;

    for (int i = 0; i < (int)pages.size(); i++) {
        int current_page = pages[i];

        // If page not in memory (page fault)
        if (find(memory.begin(), memory.end(), current_page) == memory.end()) {
            pageFaults++;

            // If memory is full, need to evict a page
            if ((int)memory.size() == frames) {
                int farthest_use_time = -1; // Stores the farthest future index
                int page_to_replace_idx = -1; // Index in 'memory' vector of page to replace

                // Iterate through pages currently in memory
                for (int j = 0; j < (int)memory.size(); j++) {
                    int mem_page = memory[j];
                    int next_use_in_future = INT_MAX; // Assume not used again

                    // Scan the future part of the reference string
                    for (int k = i + 1; k < (int)pages.size(); k++) {
                        if (pages[k] == mem_page) {
                            next_use_in_future = k; // Found next use
                            break;
                        }
                    }

                    // If this page's next use is farther than current farthest
                    if (next_use_in_future > farthest_use_time) {
                        farthest_use_time = next_use_in_future;
                        page_to_replace_idx = j;
                    }
                }
                // Replace the identified page
                memory[page_to_replace_idx] = current_page;
            } else {
                // Memory not full, just add the page
                memory.push_back(current_page);
            }
        }
        // If page is in memory (hit), do nothing for Optimal
    }
    return pageFaults;
}

int main() {
    int n, frames;
    cout << "Enter number of pages in reference string: ";
    cin >> n;

    vector<int> pages(n);
    cout << "Enter page reference string (space separated): ";
    for (int i = 0; i < n; i++) cin >> pages[i];

    cout << "Enter number of frames (physical memory capacity): ";
    cin >> frames;

    cout << "\n--- Page Replacement Results ---\n";
    cout << "FIFO    Page Faults: " << fifo(pages, frames) << endl;
    cout << "LRU     Page Faults: " << lru(pages, frames) << endl;
    cout << "Optimal Page Faults: " << optimal(pages, frames) << endl;

    return 0;
}
```
